# RB-006 네트워크 연결 장애 대응 Runbook

## Category
Network

## Symptoms
- 유선 또는 무선 네트워크가 연결되지 않는다.
- 특정 구역에서 연결이 반복적으로 끊기거나 속도가 급격히 저하된다.
- IP 주소, DNS, 게이트웨이 관련 오류가 발생한다.

## Required Information
- 영향 사용자 수, 위치, 유선/무선 구분
- SSID 또는 네트워크 포트 정보
- IP, 게이트웨이, DNS 할당 상태
- 최초 발생 시각과 최근 변경 사항
- 특정 서비스만 실패하는지 전체 통신이 실패하는지

## Resolution Steps
1. 네트워크 모니터링과 공지된 회선/장비 장애를 확인한다.
2. 단말의 링크 상태, 무선 신호, 비행기 모드와 어댑터 상태를 확인한다.
3. IP와 DNS 설정을 갱신하고 게이트웨이까지 통신을 시험한다.
4. 다른 단말 또는 포트에서 재현해 단말과 구간 문제를 분리한다.
5. 특정 구역의 다수 사용자 이슈면 AP, 스위치 포트, 오류율을 확인한다.
6. 시간대와 패킷 손실 결과를 기록해 네트워크팀에 전달한다.

## Verification
- 게이트웨이, 내부 DNS, 승인된 외부 주소로 통신되는지 확인한다.
- 연결 안정성과 지연 시간이 정상 범위로 회복됐는지 확인한다.

## Escalation Conditions
- 동일 위치 또는 구간의 여러 사용자가 영향을 받는다.
- 스위치/AP/회선 장애, 높은 패킷 손실, 반복 flap이 확인된다.
- 네트워크 장비 설정 변경이 필요하다.
