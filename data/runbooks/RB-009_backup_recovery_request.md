# RB-009 백업 및 복구 요청 Runbook

## Category
Backup / Recovery

## Symptoms
- 삭제 또는 손상된 파일/데이터베이스의 복구가 필요하다.
- 예약 백업이 실패하거나 최근 복구 지점이 보이지 않는다.
- 복구 후 데이터 무결성 확인이 필요하다.

## Required Information
- 대상 시스템, 데이터 경로, 데이터 소유자
- 손실/변경 시각과 필요한 복구 시점
- 영향 범위와 업무 중요도
- 최근 백업 job 상태와 보존 정책
- 복구 승인과 복구 대상 환경

## Resolution Steps
1. 추가 덮어쓰기를 막고 대상 데이터의 현재 상태를 보존한다.
2. 백업 catalog에서 사용 가능한 복구 지점과 job 성공 여부를 확인한다.
3. 데이터 소유자와 복구 시점, 허용 가능한 손실 범위를 합의한다.
4. 가능하면 격리된 위치에 먼저 복구해 무결성과 악성 여부를 검사한다.
5. 승인 후 운영 위치에 복원하고 접근 권한을 확인한다.
6. 복구 시간, 사용한 복구 지점, 검증 결과를 기록한다.

## Verification
- 데이터 소유자가 파일 또는 서비스의 정상 동작을 확인한다.
- 백업 job이 다시 정상 수행되고 다음 복구 지점이 생성되는지 확인한다.

## Escalation Conditions
- 유효한 복구 지점이 없거나 백업이 연속 실패했다.
- 핵심 데이터의 손실이 진행 중이거나 RPO/RTO 위반 위험이 있다.
- 랜섬웨어 또는 의도하지 않은 대량 삭제가 의심된다.
