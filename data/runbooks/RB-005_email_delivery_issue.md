# RB-005 이메일 송수신 장애 대응 Runbook

## Category
Email

## Symptoms
- 메일이 발송되지 않거나 수신자에게 도착하지 않는다.
- 특정 도메인 또는 여러 사용자의 메일이 지연된다.
- 반송 메시지, 용량 초과, 스팸 격리 알림이 발생한다.

## Required Information
- 발신자, 수신자 도메인, 발생 시각
- 메일 제목과 message trace에 필요한 식별 정보
- 반송 코드 또는 오류 메시지
- 단일/다수 사용자 및 내부/외부 메일 여부
- 첨부파일 크기와 파일 유형

## Resolution Steps
1. 메일 서비스 상태와 공지된 장애를 확인한다.
2. 사서함 용량, 라이선스, 계정 상태를 확인한다.
3. 스팸 격리, 전송 규칙, 차단 목록을 점검한다.
4. message trace로 전달 경로와 실패 지점을 확인한다.
5. 클라이언트 문제면 웹메일에서 재현 여부와 동기화 상태를 확인한다.
6. 외부 도메인 문제면 반송 코드와 상대 서버 응답을 기록한다.

## Verification
- 내부 및 허용된 외부 수신자에게 테스트 메일이 전달되는지 확인한다.
- message trace에서 최종 상태가 delivered인지 확인한다.

## Escalation Conditions
- 여러 사용자의 메일 흐름이 중단되거나 장시간 지연된다.
- 전송 규칙 변경, 도메인 평판, 보안 정책 문제가 의심된다.
- 메시지가 소실되었거나 관리자 추적이 필요하다.
