# RB-008 의심 로그인 경보 대응 Runbook

## Category
Security

## Symptoms
- 사용자가 알 수 없는 위치 또는 단말의 로그인 알림을 받았다.
- 짧은 시간에 다수의 로그인 실패나 MFA 요청이 발생한다.
- 계정에서 의도하지 않은 메일 규칙 또는 세션이 발견된다.

## Required Information
- 대상 계정과 경보 발생 시각
- 출발 위치, IP, 단말, 애플리케이션
- 사용자의 본인 활동 여부
- 성공/실패 로그인과 MFA 승인 여부
- 계정에서 발견된 변경 사항

## Resolution Steps
1. 사용자에게 의심 활동의 본인 여부를 별도 채널로 확인한다.
2. 의심 세션과 refresh token을 취소하고 계정을 임시 보호한다.
3. 안전한 단말에서 비밀번호를 변경하고 MFA 수단을 검토한다.
4. 로그인 기록, 메일 규칙, 권한 변경과 다운로드 활동을 보존한다.
5. 보안팀에 시간순 증거와 수행한 containment 조치를 전달한다.
6. 영향 범위가 확인될 때까지 관련 계정과 자산을 모니터링한다.

## Verification
- 승인되지 않은 세션과 MFA 수단이 제거됐는지 확인한다.
- 새 자격 증명으로 정상 로그인되고 추가 경보가 없는지 확인한다.

## Escalation Conditions
- 성공한 의심 로그인, MFA 오승인, 데이터 접근 흔적이 있다.
- 여러 계정 또는 관리자 계정이 영향을 받는다.
- 악성 규칙, token 탈취, lateral movement 가능성이 있다.
