# RB-001 VPN 접속 장애 대응 Runbook

## Category
VPN

## Symptoms
- VPN 인증 후 연결이 즉시 끊기거나 반복해서 재연결된다.
- 특정 사용자 또는 여러 원격 근무자가 사내 자원에 접속하지 못한다.
- VPN 클라이언트에 인증, 터널, 게이트웨이 관련 오류가 표시된다.

## Required Information
- 영향 사용자 수와 최초 발생 시각
- 사용자 위치와 사용 중인 네트워크
- 단말 OS, VPN 클라이언트 버전, 오류 메시지
- MFA 성공 여부와 최근 비밀번호/인증서 변경 여부
- 다른 인터넷 서비스의 정상 동작 여부

## Resolution Steps
1. 공지된 VPN 장애나 유지보수 여부와 게이트웨이 상태를 확인한다.
2. 사용자의 일반 인터넷 연결과 시스템 시간을 확인한다.
3. VPN 클라이언트를 완전히 종료한 뒤 최신 정책으로 다시 연결한다.
4. 저장된 VPN 프로필과 자격 증명을 갱신하고 MFA를 다시 수행한다.
5. 다른 네트워크에서도 재현되는지 확인해 로컬 회선 문제를 분리한다.
6. 다수 사용자 이슈면 공통 OS, 클라이언트 버전, 게이트웨이를 비교한다.

## Verification
- VPN 연결이 10분 이상 유지되는지 확인한다.
- 승인된 사내 서비스에 접속하고 DNS 조회가 정상인지 확인한다.

## Escalation Conditions
- 동일 시간대에 여러 사용자가 영향을 받는다.
- 게이트웨이 오류, 인증서 오류 또는 터널 생성 실패가 지속된다.
- 표준 재연결 후에도 다른 네트워크에서 동일하게 재현된다.
