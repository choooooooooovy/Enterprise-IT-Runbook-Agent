# RB-002 MFA 인증 실패 대응 Runbook

## Category
MFA

## Symptoms
- 인증 앱 알림이 오지 않거나 승인 후에도 로그인이 실패한다.
- 등록한 전화번호 또는 인증 단말을 사용할 수 없다.
- 반복 인증 요청이나 MFA 등록 요구가 표시된다.

## Required Information
- 대상 계정과 영향 서비스
- 등록된 MFA 방식과 사용 가능한 대체 방식
- 오류 메시지, 발생 시각, 마지막 성공 인증 시각
- 단말 교체 또는 전화번호 변경 여부
- 계정 잠금 및 위험 로그인 알림 여부

## Resolution Steps
1. 계정 잠금, 서비스 장애, 위험 로그인 차단 여부를 확인한다.
2. 단말 시간과 네트워크 연결을 확인하고 인증 앱을 다시 연다.
3. 등록된 대체 인증 방식으로 본인 확인을 시도한다.
4. 조직의 본인 확인 절차를 완료한 경우 MFA 세션 또는 등록을 초기화한다.
5. 사용자가 새 단말에 인증 수단을 재등록하도록 안내한다.
6. 반복 요청이면 활성 세션을 종료하고 계정 보안 상태를 점검한다.

## Verification
- 새 세션에서 MFA 인증과 대상 서비스 로그인이 모두 성공하는지 확인한다.
- 등록된 인증 수단이 사용자 의도와 일치하는지 확인한다.

## Escalation Conditions
- 본인 확인이 불가능하거나 승인되지 않은 MFA 수단이 발견된다.
- 여러 사용자에게 알림 지연 또는 인증 실패가 발생한다.
- 위험 로그인 또는 MFA 피로 공격이 의심된다.
